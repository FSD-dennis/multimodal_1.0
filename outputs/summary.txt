========================================================================
REFLECTIONS: WHAT WORKED, WHAT FAILED, WHAT TO IMPROVE
========================================================================

## What Worked
- Pipeline ran end-to-end: data collection → features → modelling → eval.
- Strict chronological split prevents any look-ahead bias.
- FinBERT provides nuanced sentiment beyond simple positive/negative.
- Loughran-McDonald lexicon captures domain-specific language.
- Price features (momentum, vol, gap) are standard alpha signals.

- direction_1d_logistic: accuracy = 56.2% (above random baseline)
- direction_1d_random_forest: accuracy = 60.4% (above random baseline)
- direction_1d_gradient_boosting: accuracy = 54.2% (above random baseline)
- direction_3d_logistic: accuracy = 58.3% (above random baseline)
- direction_3d_random_forest: accuracy = 50.0% (at or below random baseline)
- direction_3d_gradient_boosting: accuracy = 50.0% (at or below random baseline)

## What Did Not Work Well
- Small sample size (~32 events for 8 tickers × 4 quarters) limits model generalisation.
- 8-K filings may not contain rich earnings commentary for all events.
- FinBERT is trained on general financial text; earnings-specific fine-tuning could help.
- Logistic/Ridge regression cannot capture non-linear feature interactions.
- Post-earnings drift is notoriously noisy; a single quarter's data is insufficient.

## Recommended Improvements for v2
1. Expand data to 3–5 years of quarterly earnings for 20+ tickers.
2. Add earnings-call transcripts (not just SEC filings) for richer text signal.
3. Fine-tune FinBERT on earnings-specific labelled data.
4. Try gradient-boosted trees (XGBoost/LightGBM) for non-linear fusion.
5. Add intraday features (bid-ask spread, order flow imbalance) for microstructure signal.
6. Implement a transformer-based multimodal fusion architecture (cross-attention over text + price embeddings).
7. Use an LLM (via the LLMAdapterInterface) for chain-of-thought reasoning about management tone.
8. Implement walk-forward validation instead of single split.
